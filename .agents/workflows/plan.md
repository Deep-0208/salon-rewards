# Purpose

Understand the project before implementation.

Do not generate code.

Read first:

- docs/02_prd.md
- docs/03_ui_spec.md
- docs/05_build_rules.md
- docs/06_build_spec.md

If UI related:

- skills/frontend-design/SKILL.md
- docs/design.md

Process:

## Step 1 — Understand

Read project documentation.

Extract:

- Product goal
- Main user flows
- Constraints
- Locked decisions
- Technical assumptions

---

## Step 2 — Analyze Structure

Inspect:

- Existing folders
- Existing components
- Existing dependencies
- Existing architecture

Do not modify files.

---

## Step 3 — Risk Review

Identify:

- Scope creep risk
- Duplicate logic
- Architecture conflicts
- Missing requirements
- Performance concerns

---

## Step 4 — Build Order

Return:

Phase 1
Files to touch

Phase 2
Files to touch

Phase 3
Files to touch

Build incrementally.

---

## Step 5 — Recommend Next Task

Return:

Task Name

Goal

Files involved

Acceptance criteria

Restrictions:

- No code
- No file creation
- No redesign
- No architecture changes
- No future phases

Output format:

# Understanding

# Risks

# Build Order

# Recommended Next Task
